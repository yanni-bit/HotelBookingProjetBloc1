Responsive Design - Hôtel Booking

Introduction

Ce document détaille l'approche responsive mise en œuvre dans le projet Hôtel Booking pour garantir une expérience utilisateur optimale sur tous les types d'appareils (smartphones, tablettes, ordinateurs de bureau). Le responsive design est une exigence du référentiel RNCP Bloc 1 (Critère C1.b).

Approche générale

Stratégie Mobile-First

Principe : Les styles de base sont conçus pour les mobiles, puis enrichis progressivement pour les écrans plus larges.

Avantages :

  • Priorité au mobile (statistiques d'usage actuelles)
  • Performance optimisée (chargement progressif)
  • Cascade CSS naturelle
  • Maintenance facilitée

Implémentation :

/* Styles de base : mobile */
.element {
  font-size: 14px;
  padding: 0.5rem;
}

/* Enrichissement pour tablette */
@media (min-width: 768px) {
  .element {
    font-size: 16px;
    padding: 1rem;
  }
}

/* Enrichissement pour desktop */
@media (min-width: 992px) {
  .element {
    font-size: 18px;
    padding: 1.5rem;
  }
}

Techniques utilisées

  1. Media queries CSS : Adaptations par taille d'écran.
  2. Système de grille Bootstrap : Organisation du contenu en colonnes.
  3. Classes utilitaires Bootstrap : Affichage/masquage d'éléments selon le breakpoint.
  4. Flexbox et Grid CSS : Mise en page avancée et réorganisation des éléments.
  5. Unité de mesure relatives : Utilisation de rem, em, % et vw.
  6. window.matchMedia() JavaScript : Réorganisation complexe du DOM pour les grands écrans (voir ANNEXE_MATCHMEDIA.md).

Breakpoints du projet

Le projet utilise la grille Bootstrap v5 par défaut, enrichie de classes utilitaires et de media queries personnalisées. Les breakpoints sont basés sur les standards de l'industrie :

Règle générale : Le projet est conçu pour être fluide (s'adapte entre les breakpoints) et non statique (fixe aux breakpoints).

Taille Breakpoint (CSS) Utilisation typique
Extra Small (xs) < 576px Smartphones (portraits)
Small (sm) ≥ 576px Smartphones (paysage)
Medium (md) ≥ 768px Tablettes (portrait)
Large (lg) ≥ 992px Tablettes (paysage) / Petits Desktops
Extra Large (xl) ≥ 1200px Desktops standards
Extra Extra Large (xxl) ≥ 1400px Grands écrans / TV

Configuration des media queries (CSS)

Exemple (style.css) :

/* Masquer le texte du bouton sur les petits écrans */
@media (max-width: 767px) {
  .btn-avec-texte .btn-text {
    display: none;
  }
}

/* Réorganiser le formulaire de recherche sur les tablettes */
@media (min-width: 768px) and (max-width: 991px) {
  .search-form {
    flex-direction: column;
  }
}

Réorganisation du DOM avec JavaScript (matchMedia)

Pour certaines réorganisations complexes impossibles en CSS pur (comme déplacer un élément dans une structure DOM différente), l'API native window.matchMedia() est utilisée.

Cas d'usage : Modèle de chambre sur mobile

Sur mobile, l'information de prix doit être déplacée à côté du bouton de réservation. Sur desktop, elle doit rester dans la colonne d'information.

Code JavaScript (responsive.js) :

const mqMobile = window.matchMedia('(max-width: 767px)');
const priceElement = document.getElementById('room-price');
const targetMobile = document.getElementById('reservation-button-wrapper');
const targetDesktop = document.getElementById('info-column-summary');

function handlePricePosition(e) {
  if (e.matches) {
    // Mobile : déplacer le prix à côté du bouton
    targetMobile.prepend(priceElement);
  } else {
    // Tablette/Desktop : restaurer le prix dans la colonne d'info
    targetDesktop.appendChild(priceElement);
  }
}

// Lancement initial
handlePricePosition(mqMobile);

// Écoute des changements de breakpoint
mqMobile.addEventListener('change', handlePricePosition);

Avantages de matchMedia :

  • API native et performante
  • Évite l'écoute gourmande en ressources de l'événement window.resize
  • Code asynchrone pour les changements de média (meilleure performance)

Gestion des images et médias

Images fluides (Fluid Images)

Toutes les images sont fluides et respectent leur conteneur :

Implémentation CSS :

/* Règle générale Bootstrap (ou équivalent) */
img {
  max-width: 100%; /* L'image ne dépasse jamais son conteneur */
  height: auto;   /* Préserve le ratio */
  display: block; /* Évite les marges sous les images */
}

/* Images décoratives non critiques */
.img-lazy {
  object-fit: cover;
  width: 100%;
  height: 100%;
}

Icônes et graphiques

Utilisation d'icônes vectorielles (SVG via Bootstrap Icons) pour garantir une qualité d'affichage parfaite et une taille de fichier minimale sur tous les niveaux de zoom et toutes les résolutions (Retina).

Tests de validation responsive

Les tests manuels et automatisés sont effectués sur les critères suivants :

Mise en page générale :

  • Pas de scroll horizontal (sauf en cas de besoin absolu, ex: grandes tables de données).
  • Éléments correctement alignés et empilés.

Navigation :

  • Menu hamburger fonctionnel sur mobile.
  • Liens cliquables et zones tactiles adaptées (44px min).

Images :

  • Pas de débordement
  • Ratio préservé
  • Chargement correct

Formulaires :

  • Champs de taille appropriée
  • Labels visibles
  • Validation claire

Typographie :

  • Lisibilité sur tous écrans
  • Taille adaptée
  • Line-height confortable

Espacements :

  • Marges cohérentes
  • Padding adapté
  • Pas de chevauchement

Conclusion

Le projet Hôtel Booking implémente une approche responsive complète combinant :

  • Stratégie mobile-first
  • Grille Bootstrap 12 colonnes
  • Media queries CSS custom (18 breakpoints)
  • Classes utilitaires Bootstrap
  • MatchMedia JavaScript pour réorganisation DOM
  • Images et typographie responsive
  • Support multi-appareils (mobile, tablette, desktop)

Cette approche garantit une expérience utilisateur optimale sur tous les types d'appareils, conformément aux exigences du référentiel RNCP Bloc 1 (Critère C1.b).
